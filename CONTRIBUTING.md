# Ghid de Contribuție / Contributing Guide

## 🇷🇴 Română

### Cum să contribui

1. **Fork** repository-ul
2. Creează un **branch** pentru feature-ul tău: `git checkout -b feature/adauga-psalmi`
3. **Commit** modificările: `git commit -m 'feat: adaugă Psalmi complet'`
4. **Push** pe branch: `git push origin feature/adauga-psalmi`
5. Deschide un **Pull Request**

### Standarde de cod

- TypeScript strict (`"strict": true` în tsconfig)
- ESLint + Prettier (configurate în proiect)
- Teste Jest pentru logica backend (target: 80% coverage)
- Commits în format [Conventional Commits](https://www.conventionalcommits.org/)

### Tipuri de contribuții

- 📚 **Date biblice** – Adăugare cărți/capitole/versete în JSON
- 🤖 **Prompt engineering** – Îmbunătățire YAML prompts hermeneutice
- 🎨 **UI/UX** – Îmbunătățiri vizuale Angular
- 🔧 **Backend** – Noi endpoint-uri, performanță
- 🌍 **Localizare** – Traduceri interfață

## 🇬🇧 English

### How to contribute

1. **Fork** the repository
2. Create a **branch**: `git checkout -b feature/add-psalms`
3. **Commit** your changes: `git commit -m 'feat: add complete Psalms'`
4. **Push** to the branch: `git push origin feature/add-psalms`
5. Open a **Pull Request**

### Code standards

- TypeScript strict mode
- ESLint + Prettier (project-configured)
- Jest tests for backend logic (target: 80% coverage)
- [Conventional Commits](https://www.conventionalcommits.org/) format

### Contribution areas

- 📚 **Biblical data** – Add books/chapters/verses to JSON
- 🤖 **Prompt engineering** – Improve hermeneutic YAML prompts
- 🎨 **UI/UX** – Angular visual improvements
- 🔧 **Backend** – New endpoints, performance
- 🌍 **Localization** – UI translations

*This project follows principles of mutual respect and theological academic dialogue.
